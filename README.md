## Why work with me

- **WebApp focus, not spray-and-pray**: methodical coverage of authn/z, input/output, storage, caching, file handling, and integration boundaries.  
- **OSINT-driven recon** that uncovers legacy endpoints, shadow assets, and real attack paths.  
- **Deliverables that ship**: PoC with replay steps, risk mapping per component, prioritized remediation, and **re-test** to close the loop.

---

## Technical Scope

### Application Security (Pentest)
- **OWASP Top 10 / ASVS**: SQLi/NoSQLi, XSS (reflected/stored/DOM), IDOR/BOLA, Path Traversal, CSRF, SSRF basics, insecure deserialization.
- **Auth & Session**: password reset flaws, session fixation, token storage/rotation, cookie flags, OAuth/OIDC misconfig.
- **Business Logic**: role matrix abuse, rate limiting, enum/guess, mass assignment.

### OSINT & Recon
- Subdomain & asset discovery (amass, massdns, ffuf), tech fingerprinting, archived routes (**Wayback**), metadata & breach intel correlation.
- People/org OSINT (Maltego, Sherlock) and **attack surface** consolidation.

### Reverse & Mobile (targeted)
- APK decompilation, basic hook scenarios, TLS interception, malicious payload analysis.  
- IOC extraction & capability mapping.

### Infrastructure for Testing
- Labs on **Proxmox** / **VirtualBox**, segmentation via **Ubiquiti** (DMZ/NAT), **T-POT** honeypot isolation.  
- Linux administration (Fedora / Ubuntu / Debian), automation in **Bash** & **Python**.

---

## Methodology (concise & reproducible)

1. **Threat Modeling & Scope** — assets, trust boundaries, data classes.  
2. **OSINT & Recon** — DNS, subdomains, archived routes, tech stack.  
3. **Surface Mapping** — authZ matrices, parameters inventory, file paths.  
4. **Exploitation** — manual-first with tool assistance; impact-oriented PoC.  
5. **Abuse Case Validation** — privilege pivoting, data exfil routes, persistence.  
6. **Reporting & Retest** — CVSS/likelihood, fix strategy, validation.

<details>
  <summary><b>Operational Checklists (open)</b></summary>

- **AuthN/Z**: session lifecycle, token scope, RBAC/ABAC, IDOR matrices per role.  
- **Input/Output**: context-aware encoding, SSTI/TTI baselines, XXE/LDAP/XPath patterns.  
- **Storage/Secrets**: hard-coded keys, backup exposure, bucket policies.  
- **Caching/CDN**: cache poisoning candidates, vary headers, key calc.  
- **File & Uploads**: content-type confusion, polyglots, traversal to secret mounts.  
</details>
